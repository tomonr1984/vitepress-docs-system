# テンプレートシステム

このガイドでは、手動でのドキュメント作成や特別なカスタマイズが必要な場合のテンプレートシステムについて説明します。

> **注意**: 通常のドキュメント生成には、[はじめに](./getting-started.md)で紹介しているカスタムコマンドをご利用ください。テンプレートシステムは学習目的や特別な要件がある場合にのみご使用ください。

## 📚 テンプレートシステムとは

テンプレートシステムは、手動でドキュメントを作成する際の雛形を提供します。以下のような場合に有用です：

- **学習目的**: ドキュメント構造を理解したい場合
- **特別なカスタマイズ**: 標準的なコマンドでは対応できない特殊な要件
- **段階的な作成**: 少しずつドキュメントを作成したい場合

## 🗂️ テンプレート構造

```
templates/
└── tool-template/
    ├── ja/          # 日本語テンプレート
    │   ├── index.md
    │   ├── installation.md
    │   ├── usage.md
    │   ├── api.md
    │   └── troubleshooting.md
    └── en/          # 英語テンプレート
        ├── index.md
        ├── installation.md
        ├── usage.md
        ├── api.md
        └── troubleshooting.md
```

## 🛠️ 手動でのテンプレート使用方法

### ステップ1: ディレクトリ作成

```bash
# ツール用のディレクトリを作成
mkdir -p docs/tools/[tool-name] docs/en/tools/[tool-name]
```

### ステップ2: テンプレートのコピー

```bash
# 日本語テンプレートをコピー
cp templates/tool-template/ja/* docs/tools/[tool-name]/

# 英語テンプレートをコピー
cp templates/tool-template/en/* docs/en/tools/[tool-name]/
```

### ステップ3: 内容の編集

各ファイルを編集して、ツール固有の情報に置き換えます：

- **index.md**: ツールの概要と主要機能
- **installation.md**: インストール手順
- **usage.md**: 基本的な使用方法とワークフロー
- **api.md**: コマンドや関数の詳細仕様
- **troubleshooting.md**: よくある問題と解決法

### ステップ4: 設定の更新

1. **ナビゲーション設定**: `docs/.vitepress/config.ts`にツールを追加
2. **インデックス更新**: メインページにツールのリンクを追加

## 📖 ドキュメンテーション標準

すべてのツールは以下の**5セクション構造**で統一されています：

1. **概要 (Overview)** - ツールの目的と主要機能
2. **インストール (Installation)** - セットアップ手順
3. **使用ガイド (Usage Guide)** - 基本的な使い方とワークフロー
4. **APIリファレンス (API Reference)** - 完全なコマンド仕様
5. **トラブルシューティング (Troubleshooting)** - よくある問題と解決法

詳細な標準については、[ドキュメンテーション標準](../documentation-standards)を参照してください。

## 🎯 推奨事項

- **まずはコマンドを試す**: 自動生成コマンドで十分な場合がほとんどです
- **段階的な学習**: テンプレートで構造を理解してから自動化に移行
- **一貫性の維持**: 標準的な構造から大きく逸脱しないようにする
- **品質の確保**: 他のドキュメントと同じレベルの品質を保つ
